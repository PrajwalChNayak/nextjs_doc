⚡ getStaticProps

Pre-build your pages with data at build time! Perfect for blogs, product pages, and any content that doesn't change often. Super fast and SEO-friendly! 🚀

🤔 What Does getStaticProps Do?

⏰ When It Runs

At build time - not when users visit your page! This means your pages are pre-generated with all the data ready to go.

🎯 Perfect For

  • • Blog posts and articles
  • • Product catalogs
  • • Marketing pages
  • • Documentation sites
💡

The Magic:

Your data is fetched once during build, then served instantly to every visitor. No waiting for API calls!

export async function getStaticProps() {
  // This runs at build time, not on every request!
  console.log('🏗️ Fetching data at build time...')
  
  // Fetch data from any source
  const res = await fetch('https://jsonplaceholder.typicode.com/posts')
  const posts = await res.json()
  
  // Take only first 10 posts for demo
  const limitedPosts = posts.slice(0, 10)

  return {
    props: {
      posts: limitedPosts, // This gets passed to your component
    },
  }
}

🔄 Advanced: Keep Your Data Fresh (ISR)

🤔 What if your data changes?

Sometimes you want the speed of static generation but your data changes occasionally. That's where Incremental Static Regeneration (ISR) comes in! 

export async function getStaticProps() {
  const res = await fetch('https://api.example.com/posts')
  const posts = await res.json()

  return {
    props: {
      posts,
    },
    // 🔄 Regenerate the page:
    // - At most once every 60 seconds
    // - Only when someone visits the page
    // - If the data has changed
    revalidate: 60, // seconds
  }
}

🎯 When Should You Use getStaticProps?

✅ Perfect For

  • • Blog posts and articles
  • • Product catalogs
  • • Documentation pages
  • • Marketing/landing pages
  • • Data that doesn't change often

❌ Not Great For

  • • User-specific data
  • • Real-time data (stock prices, etc.)
  • • Data that changes every second
  • • Admin dashboards
  • • Personalized content
