CSS Modules

Next.js has built-in support for CSS Modules using the [name].module.css file naming convention.

Creating a CSS Module

/* styles/Home.module.css */
.container {
  padding: 0 2rem;
}

.main {
  min-height: 100vh;
  padding: 4rem 0;
  flex: 1;
  display: flex;
  flex-direction: column;
  justify-content: center;
  align-items: center;
}

.title {
  margin: 0;
  line-height: 1.15;
  font-size: 4rem;
  text-align: center;
}

Benefits of CSS Modules

  • Scoped styles: Class names are automatically scoped to the component
  • No naming conflicts: You can use the same class name in different modules
  • Tree shaking: Unused styles are removed in production
  • TypeScript support: Get autocomplete for class names
