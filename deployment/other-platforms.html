🛠️ Quick Setup Guide

Step 1 Configure Static Export

First, tell Next.js to generate static files:

/** @type {import('next').NextConfig} */
const nextConfig = {
  output: 'export',
  trailingSlash: true,
  images: {
    unoptimized: true
  }
}

module.exports = nextConfig

Step 2 Create Netlify Configuration

Add a netlify.toml file for build settings:

[build]
  command = "npm run build && npm run export"
  publish = "out"

[[redirects]]
  from = "/*"
  to = "/index.html"
  status = 200

Step 3 Deploy to Netlify

🌐 Via Website
  1. Go to netlify.com
  2. Connect your Git repo
  3. Deploy automatically!
💻 Via CLI
npm install -g netlify-cli
netlify deploy --prod --dir=out

🛠️ Quick Deploy Guide

Step 1 Install Amplify CLI

Get the Amplify command-line tools:

npm install -g @aws-amplify/cli
amplify configure

Step 2 Create Build Configuration

Add amplify.yml for build settings:

version: 1
frontend:
  phases:
    preBuild:
      commands:
        - npm install
    build:
      commands:
        - npm run build
  artifacts:
    baseDirectory: .next
    files:
      - '**/*'
  cache:
    paths:
      - node_modules/**/*
      - .next/cache/**/*

Step 3 Deploy to Amplify

🌐 Via Console
  1. Visit AWS Amplify Console
  2. Connect your Git repository
  3. Configure build settings
  4. Deploy automatically!
💻 Via CLI
amplify init
amplify add hosting
amplify publish

🎯 3-Step Heroku Deployment

Step 1 Configure Your Package.json

Tell Heroku how to build and run your app:

{
  "scripts": {
    "dev": "next dev",
    "build": "next build",
    "start": "next start -p $PORT",
    "heroku-postbuild": "npm run build"
  }
}

Step 2 Deploy Your App

Use the Heroku CLI to create and deploy your app:

heroku create your-app-name
git push heroku main

Step 3 View Your Deployed App

Open your app in the browser:

heroku open

🐳 Docker Deployment - Containerize Your App

Flexible and portable! Docker allows you to package your Next.js app and all its dependencies into a container, ensuring consistent performance across different environments. 📦

📦

Containerization

Package your app with all dependencies

🔄

Environment Consistency

Same environment from development to production

⚙️

Easy Integration

Works with any cloud provider or on-premises

🐳 Docker Deployment Guide

Step 1 Install Docker

Follow the official Docker installation guide:

https://docs.docker.com/get-docker/

Step 2 Create a Dockerfile

Define your app's environment and dependencies:

FROM node:16-alpine AS deps
WORKDIR /app
COPY package.json package-lock.json ./
RUN npm ci --only=production

FROM node:16-alpine AS builder
WORKDIR /app
COPY package.json package-lock.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM node:16-alpine AS runner
WORKDIR /app
ENV NODE_ENV production

RUN addgroup --system --gid 1001 nodejs
RUN adduser --system --uid 1001 nextjs

COPY --from=builder /app/public ./public
COPY --from=builder /app/package.json ./package.json
COPY --from=builder --chown=nextjs:nodejs /app/.next/standalone ./
COPY --from=builder --chown=nextjs:nodejs /app/.next/static ./.next/static

USER nextjs
EXPOSE 3000
ENV PORT 3000

CMD ["node", "server.js"]

Step 3 Build and Run Your Container

Use Docker commands to build and run your container:

docker build -t nextjs-app .
docker run -p 3000:3000 nextjs-app

📦 Static Export Guide

Step 1 Update next.config.js

Configure Next.js for static export:

/** @type {import('next').NextConfig} */
const nextConfig = {
  output: 'export',
  trailingSlash: true,
  images: {
    unoptimized: true
  }
}

module.exports = nextConfig

Step 2 Run the Export Command

Generate the static files:

next export

Step 3 Deploy the Static Files

Upload the files to your hosting provider:

rsync -avz ./out/ user@your-server:/path/to/your/site
