Welcome to Next.js

The React Framework for Production

Getting Started

Next.js gives you the best developer experience with all the features you need for production: hybrid static & server rendering, TypeScript support, smart bundling, route pre-fetching, and more.

Installation

To create a new Next.js application, you can use create-next-app which sets up everything automatically for you.

npx create-next-app@latest my-app

For TypeScript projects:

npx create-next-app@latest my-app --typescript

Project Setup

After creating your project, navigate to the directory and start the development server:

cd my-app
npm run dev

Open http://localhost:3000 with your browser to see the result.

Note

You can edit pages/index.js and see the changes update in real-time thanks to Fast Refresh.

Your First App

Let's create your first page. Next.js uses a file-system based router built on the concept of pages.

Creating a Page

Create a file called about.js in the pages directory:

// pages/about.js

Now you can access this page at http://localhost:3000/about.

Pages

In Next.js, a page is a React Component exported from a .js, .jsx, .ts, or .tsx file in the pages directory. Each page is associated with a route based on its file name.

Example: If you create pages/about.js that exports a React component like below, it will be accessible at /about.

function About() {
  return 
About

}

export default About

Dynamic Routes

Next.js supports pages with dynamic routes. For example, if you create a file called pages/posts/[id].js, then it will be accessible at posts/1, posts/2, etc.

// pages/posts/[id].js
import { useRouter } from 'next/router'

const Post = () => {
  const router = useRouter()
  const { id } = router.query

  return 
Post: {id}

}

export default Post

API Routes

API routes provide a solution to build your API with Next.js. Any file inside the folder pages/api is mapped to /api/* and will be treated as an API endpoint instead of a page.

// pages/api/hello.js
export default function handler(req, res) {
  res.status(200).json({ name: 'John Doe' })
}

For a request to /api/hello, this API route will return a JSON response with { name: 'John Doe' }.

Server-Side Rendering (SSR)

Server-side rendering (SSR) is a technique where pages are rendered on the server on each request. This is useful for SEO and initial page load performance.

export async function getServerSideProps(context) {
  const res = await fetch(`https://api.example.com/data`)
  const data = await res.json()

  return {
    props: { data }
  }
}

Static Site Generation (SSG)

Static Generation is the pre-rendering method that generates the HTML at build time. The pre-rendered HTML is then reused on each request.

export async function getStaticProps() {
  const res = await fetch('https://api.example.com/posts')
  const posts = await res.json()

  return {
    props: {
      posts,
    },
    revalidate: 10, // In seconds
  }
}

Incremental Static Regeneration (ISR)

ISR enables you to use static-generation on a per-page basis, without needing to rebuild the entire site.

Benefits of ISR

  • • Pages are generated on-demand
  • • Automatic static optimization
  • • No rebuild required for new pages

getStaticProps

If you export a function called getStaticProps from a page, Next.js will pre-render this page at build time using the props returned by getStaticProps.

// This function gets called at build time
export async function getStaticProps() {
  // Call an external API endpoint to get posts
  const res = await fetch('https://jsonplaceholder.typicode.com/posts')
  const posts = await res.json()

  // By returning { props: { posts } }, the Blog component
  // will receive `posts` as a prop at build time
  return {
    props: {
      posts,
    },
  }
}

Deploy to Vercel

The easiest way to deploy your Next.js app is to use the Vercel Platform from the creators of Next.js.

  1. Push your code to a Git repository (GitHub, GitLab, Bitbucket)
  2. Import your project to Vercel
  3. Vercel will automatically detect that it's a Next.js project and configure the build settings
  4. Your application is deployed!
npm i -g vercel

Best Practices

Follow these best practices to build high-quality Next.js applications.

Performance

  • • Use the Next.js Image component for optimized images
  • • Implement proper caching strategies
  • • Use dynamic imports for code splitting

SEO

  • • Use proper meta tags with next/head
  • • Implement structured data
  • • Optimize for Core Web Vitals

Security

  • • Sanitize user inputs
  • • Use HTTPS in production
  • • Implement proper authentication
