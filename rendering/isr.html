Incremental Static Regeneration (ISR)

Incremental Static Regeneration allows you to update static content after build time without rebuilding the entire site. 

export async function getStaticProps() {
  const res = await fetch('https://api.example.com/posts')
  const posts = await res.json()

  return {
    props: {
      posts,
    },
    revalidate: 10, // In seconds
  }
}

How ISR Works

  1. A request is made to a page that was statically generated at build time
  2. The cached version is served immediately
  3. After the revalidation time, the next request triggers a regeneration
  4. Once successful, the cache is updated with the new page

Best of Both Worlds

ISR combines the benefits of static generation with the flexibility of server-side rendering.
