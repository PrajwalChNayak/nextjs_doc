🌳

Tree Shaking Magic

Import only what you need! This simple change can cut your bundle size in half.

❌ Bundle Bloat (Bad)

// Imports ENTIRE lodash library (70KB+!)
import _ from 'lodash'
const result = _.debounce(myFunction, 300)

✅ Tree Shaking (Good)

// Imports only debounce function (~2KB)
import { debounce } from 'lodash'
const result = debounce(myFunction, 300)

📊 Bundle Size Monitoring

Stay ahead of bundle bloat! Set up automated monitoring to catch size increases before they impact your users. 🚨

📏

Size Limit - Your Bundle Guardian

Automatically prevent bundle size regressions in your CI/CD pipeline.

🚀 Quick Setup

Installation 
npm install --save-dev size-limit @size-limit/preset-next
package.json 
{
  "scripts": {
    "size": "size-limit",
    "size:why": "size-limit --why"
  },
  "size-limit": [
    {
      "name": "First Load JS",
      "path": ".next/static/chunks/pages/*.js",
      "limit": "128 KB"
    },
    {
      "name": "Main Bundle",
      "path": ".next/static/chunks/main-*.js", 
      "limit": "50 KB"
    },
    {
      "name": "Framework Bundle",
      "path": ".next/static/chunks/framework-*.js",
      "limit": "45 KB"
    }
  ]
}

⚙️ Advanced Configuration

🔧

Custom Webpack Configuration

Fine-tune your bundle with custom webpack settings for maximum optimization.

next.config.js - Advanced Optimization 
module.exports = {
  webpack: (config, { buildId, dev, isServer, defaultLoaders, webpack }) => {
    // Ignore heavy libraries from bundling
    config.externals = {
      ...config.externals,
      canvas: 'canvas',
      jsdom: 'jsdom'
    }

    // Add polyfills for Node.js modules when needed
    config.resolve.fallback = {
      ...config.resolve.fallback,
      fs: false,
      net: false,
      tls: false,
      crypto: false,
      stream: false,
      url: false,
      zlib: false,
      http: false,
      https: false,
      assert: false,
      os: false,
      path: false
    }

    // Split chunks more aggressively
    if (!dev && !isServer) {
      config.optimization.splitChunks.chunks = 'all'
      config.optimization.splitChunks.cacheGroups = {
        ...config.optimization.splitChunks.cacheGroups,
        vendor: {
          test: /[\\/]node_modules[\\/]/,
          name: 'vendors',
          chunks: 'all',
        },
      }
    }

    return config
  },
}

🎯 Best Practices Checklist

Your bundle optimization game plan! Follow these practices to maintain a lightning-fast app. ⚡

✅ Bundle Optimization Do's

  • 📊 Run npm run analyze weekly
  • 🚨 Set up size-limit in CI/CD pipeline
  • 🌳 Use tree-shakable imports
  • 🚀 Implement dynamic imports for large components
  • 📱 Monitor Core Web Vitals
  • 🗜️ Enable compression and use CDN

❌ Bundle Optimization Don'ts

  • 📦 Import entire libraries when you need one function
  • 🏗️ Load heavy dependencies on initial page load
  • 🤔 Ignore bundle size warnings
  • 🔄 Add multiple libraries that do the same thing
  • 🎯 Skip testing bundle sizes in staging
  • 📈 Forget to analyze after adding new dependencies
