🔌 API Routes

Build your backend API right inside your Next.js app! No separate server needed. Create endpoints for data, authentication, payments, and more! 🚀

🤔 What Are API Routes?

Think of API routes as your app's backend functions. They handle data processing, database operations, authentication, and more - all without needing a separate server!

📁 File Location

pages/
  api/
    hello.js  ← API endpoint!

🌐 URL Access

GET /api/hello

Automatically available at this URL!

👶 Your First API Route

🎯 Let's create a simple "Hello World" API! 

// pages/api/hello.js
export default function handler(req, res) {
  // Send back a JSON response
  res.status(200).json({ 
    message: 'Hello from Next.js API! 👋',
    timestamp: new Date().toISOString(),
    success: true
  })
}
🎉

Test it out!

Visit http://localhost:3000/api/hello in your browser and see the JSON response!

🔄 Handling Different HTTP Methods

🎯 Real-World Example: User Management API 

// pages/api/users.js
export default function handler(req, res) {
  const { method } = req

  switch (method) {
    case 'GET':
      // Get all users
      res.status(200).json({ 
        users: ['John', 'Jane', 'Bob'],
        message: 'Users retrieved successfully! 📋'
      })
      break
      
    case 'POST':
      // Create a new user
      const { name } = req.body
      res.status(201).json({ 
        message: `User ${name} created! 🎉`,
        user: { id: Date.now(), name }
      })
      break
      
    case 'DELETE':
      // Delete a user
      res.status(200).json({ 
        message: 'User deleted! 🗑️'
      })
      break
      
    default:
      res.setHeader('Allow', ['GET', 'POST', 'DELETE'])
      res.status(405).end(`Method ${method} Not Allowed`)
  }
}

GET Request

Retrieve data (like getting user list)

POST Request

Create new data (like adding a user)

DELETE Request

Remove data (like deleting a user)
